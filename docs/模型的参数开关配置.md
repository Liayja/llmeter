# deepseek-v4-pro如何设置推理参数

需要让模型投入更多推理时，可以在支持的模型上设置 reasoning_effort。DeepSeek V4 的 OpenAI-compatible 路径建议使用 high 或 max。如果当前请求已经设置 thinking.type: "disabled"，不要同时发送 reasoning_effort。该参数是否改变质量、延迟和 token 用量，需要用业务 prompt 验证。

```json
extra_body = {
    "thinking": {"type": "enabled"},
    "reasoning_effort": "high",
}
```

DeepSeek 推理内容通常位于 reasoning_content.

实际在压测脚本中可以这样设置：
```json
extra_params:
    thinking:
      type: disabled 
```


参考链接：https://wcode.net/model/deepseek-v4-pro
https://docs.infini-ai.com/gen-studio/api/text-generation/tutorial-reasoning/deepseek.html

# glm-5.2模型推理参数

参考链接：https://docs.infini-ai.com/gen-studio/api/text-generation/tutorial-reasoning/glm.html

reasoning_effort 只适用于 GLM-5.2。不要把这个字段直接复制到 glm-5.1、glm-5 或 glm-4.7 的请求中。

GLM-5.2 的默认推理强度是 max。如果业务希望显式控制推理强度，建议先只使用有实际区分度的取值：

high：较高推理强度。
max：最高推理强度，也是默认值。
其他兼容取值会被模型侧归并或改变思考开关语义：

none、minimal 会跳过思考；需要关闭思考时，优先使用 thinking.type: "disabled"。
low、medium 会映射为 high。
xhigh 会映射为 max。

```json
extra_body = {
    "thinking": {"type": "enabled"},
    "reasoning_effort": "max",
}
```


但是实际压测脚本中要这样写：
```json
 extra_params:
    reasoning_effort: none  
```
